# Project Roadmap

This document records the eight-phase build plan for local-coding-agent.

## Phase Zero — Design and Reference-Agent Evaluation (complete)

Four structured evaluation sessions of OpenAI's Codex CLI, covering
scaffolding, ambiguous-brief handling, multi-file refactoring, and
diagnostic bug-hunting. The patterns observed are translated into the
design requirements document at docs/design/design-requirements.md.

## Phase One — Minimal ReAct Loop (planned)

A minimal agent loop in Python against a local Ollama server.

## Phase Two — File Manipulation Tools (planned)

File reading, writing, and context-anchored patch application.

## Phase Three — Persistent Context (planned)

Session persistence and project-level configuration.

## Phase Four — Sandboxing and Approval Policies (planned)

Platform sandboxing and graduated approval modes.

## Phase Five — Multi-File Editing and Indexing (planned)

Multi-file edits and a lightweight repository index.

## Phase Six — MCP Client Support (planned)

Model Context Protocol client integration.

## Phase Seven — Textual TUI (planned)

An interactive terminal user interface.

## Phase Eight — Evaluation Harness (planned)

A benchmark harness combining public and personal evaluation tasks.
